The Concepts, Fully Explained

🎯 Prompt Harness Engineering

Think of your prompt as a harness — a structured framework that directs Claude's energy precisely. Raw prompts produce average results. Harnessed prompts define a role, context, constraints, and output format all at once.

Claude is trained to recognise XML tag structures. Wrapping sections inside <context>, <task>, and <format> tags produces far more precise responses than free-form text.

Pro Pattern — The 4-Part Harness Role: Who is Claude in this conversation?  ·  Context: What does it need to know?  ·  Task: What exactly should it do?  ·  Format: How should the output look? Never skip Format — it's where most beginners leave quality on the table.
# ✗ Bare prompt (generic output guaranteed)
"Write a product description for my coffee maker"

# ✓ Harnessed prompt (precise, repeatable output)
<role>You are a conversion copywriter specialising in DTC brands.</role>
<context>Product: AeroPress Go. Audience: remote workers 25–40. USP: brews in 60 sec.</context>
<task>Write a description that emphasises speed and portability.</task>
<format>One punchy headline (max 8 words) + two short benefit sentences. No fluff.</format>

🤖 Understanding AI Agents

An agent is Claude given tools — the ability to take real-world actions. Instead of just answering questions, an agent can read files, browse websites, write and run code, call APIs, and loop through multi-step workflows on its own.

The mental model: Agent = Claude + a loop + tools. Claude decides what to do, does it, observes the result, then decides the next step. This cycle repeats until the goal is achieved.

The 3 Laws of Good Agents 1. Give Claude a goal, not a script.  ·  2. Provide exactly the tools the goal requires.  ·  3. Define guardrails (what NOT to do) rather than micromanaging every step.
1

Define a clear, measurable goal

"Research the top 5 competitors and summarise their pricing in a table" — not "search Google for competitors."

2

Equip it with the right tools

Web search, file read/write, code execution. Only give Claude tools it actually needs — fewer tools means fewer mistakes.

3

Set guardrails, not micromanagement

Specify what Claude should NOT do ("don't make purchases, don't delete files") and let it figure out the how.

4

Review before irreversible actions

For anything that can't be undone (sending emails, database writes), have Claude confirm before proceeding.

📄 The CLAUDE.md File — Persistent Project Memory

Claude Code reads a CLAUDE.md file at the root of your project at the start of every session. This is your project's memory. Write your conventions, architecture decisions, and preferred patterns here once, and Claude will never need them explained again.

What Belongs in CLAUDE.md Project overview & folder structure  ·  Tech stack + versions  ·  Naming conventions  ·  What Claude should NEVER do  ·  Test & build commands  ·  Key architectural decisions  ·  External API notes  ·  Preferred code patterns
# Example CLAUDE.md snippet

## Project
E-commerce dashboard. Next.js 14 App Router + Tailwind + Prisma + PostgreSQL.

## Conventions
- Components: PascalCase in /components/ui/
- API routes: kebab-case in /app/api/
- Always use TypeScript strict mode. Never use `any`.

## NEVER
- Do not delete migration files
- Do not commit directly to main

## Commands
npm run dev  ·  npm run build  ·  npm run test

🗺️ Planning Mode & Task Management

For multi-step or complex tasks, making Claude plan before acting is the single biggest quality multiplier available. Use /plan in Claude Code, or ask Claude to "think through the approach first" in regular chat.

For ongoing projects, Claude Code's TodoWrite tool maintains a live task list that Claude updates as it works — so you always know what's done, what's in progress, and what's next.

Planning Prompt Template "Before you write any code, think through: (1) what files will need to change, (2) what could go wrong, (3) what you'll do first, second, and third. Write a numbered plan, then wait for my approval before starting."

🏗️ Agent Teams & Orchestration

One agent is powerful. A team of specialised agents is transformative. In an agent team, one Claude instance acts as the orchestrator — it breaks down a goal and delegates to specialised subagents: one that searches, one that writes, one that reviews, one that tests.

Orchestrator Pattern Orchestrator receives goal → breaks it into subtasks → spins up subagents with specific roles → collects results → synthesises final output. The orchestrator manages and integrates — it never does the work itself.

🧠 Context Window Management

Claude has no persistent memory between API calls. Every new message re-sends the entire conversation history from scratch. Context grows over time — and eventually old information gets pushed out or causes confusion.

Skilled users manage context proactively: use /compact before the window gets bloated, write crisp session handoff notes, and avoid re-injecting information Claude already has.

The Golden Rule of Context Before every message, ask: "Does Claude still have the context it needs, or am I re-injecting information it already has?" This habit alone will dramatically improve response quality and cut costs.
# Session handoff note template (saves tokens next session)

## Session Handoff — [Date]
What we accomplished: [2–3 bullet points]
Current state: [where the project sits right now]
Next task: [the exact first thing to do in the next session]
Blockers: [anything Claude should know upfront]

💰 Token Savings — Work Smarter, Not Longer

Tokens are the currency of Claude. Every word you send, every word Claude responds with, and every word of prior conversation costs tokens — meaning money on the API and slower responses in the UI.

Highest-leverage moves: (1) use CLAUDE.md to replace dozens of re-explanations, (2) write one complete scoped prompt instead of five back-and-forth exchanges, (3) trim files to relevant lines before pasting, (4) compact early and often, (5) use session handoff notes.

Quick Token Wins — Do These Today ✓ Create CLAUDE.md before every project session  ·  ✓ Write your full task spec before opening a session  ·  ✓ Paste only the relevant 20 lines of a file, not the full 400  ·  ✓ Run /compact after every 10–15 exchanges  ·  ✓ End each session with a handoff note for next time

Frequently Asked Questions

Got a question? We've got answers. Here are some of the most common questions about using Claude effectively.

What is the difference between Claude Sonnet and Opus?
Sonnet is faster and cheaper, making it ideal for most daily coding, writing, and agentic tasks. Opus is the heavyweight model, meant for highly complex, multi-step logic problems where raw reasoning power is prioritized over speed.
Do I need to know how to code to use Claude Agents?
Not at all! While developers can build custom agents using APIs, tools like Claude Code and other no-code platforms allow anyone to set up autonomous agents using plain English instructions and a simple CLAUDE.md file.
What exactly does the /compact command do?
The /compact command compresses your ongoing conversation history. It takes the key context and summarizes it, dropping verbatim past messages. This keeps the context window lean, reducing API costs and preventing Claude from getting confused by outdated instructions.
Is the Udemy course completely free?
Yes, the AI Prompt Engineering course listed in our videos section is a free resource designed to get you up to speed with ChatGPT, Claude, and Claude Code fast.
